Let's take a look at Microservices

Why?

  • Interesting concept, goes back to 2005
  • Attracting attention in 2011 (Netflix, Amazon)
  • Aims to solve problems of monolithic apps
  • Dr. Peter Rodgers, got used the term "Micro-Web-Services" 2005 at a Cloud Computing Conference5
  • Software Architects Conference, 2001

A MS definition by Martin Fowler:

“A particular way of designing software applications as suites of independently deployable services.”
Martin Fowler: Author, conferencista, evangelista del desarrollo de sw, fan del metodo agil. Una forma de disenar aplicaciones de sw en forma de servicios que se pueden publicar independientemente

MS characteristics

  • Based on components
  • Heterogeneous - You build it, you run it!
  • Decentralized Data Management
  • Service Independent Deployment
  • Scalability and Maintenance
  • Agrees with Twelve-Factor App
  • Servicios con funcionalidad de negocio encapsulada
  • Poca cohesion, por lo tanto pueden ser desarrados en dierentes tecnologias
  • Con interfaces bien definidas
  • Concuerda con las 12 Factores para generar applicaciones

Monolithic and Microservice Architectural Patterns

Monolithic Pros and Cons

  • Easy to develop, till it grows
  • Easy to scale and deploy, but all or nothing
  • Standard technology, but difficult to evolve

Therefore, bad architectural pattern?

Microservices Pros and Cons

  • Facilitates deployments - DevOps friendly, but complex distributed system
  • Organized independent development, but highly service interaction needed
  • Simplifies functional testing, but int tests can be cumbersome

Therefore, optimal architectural pattern?

  • Facilita la continua publicacion de versiones, manejo de transacciones complejo
  • Propio para desarrollo en equipos independientes, pero require mucha comunicacion entre ellos
  • Tests de integracion pueden ser complicados, dado que cada equipo solo ve una parte del todo

Microservices as Enterprise Applications

Real Case Microservice System

An automobile company requires to migrate a monolithic legacy system. The new system should be web-based with optimized performance and highly scalable.

  • Systema de conformidad de vehiculos para mercados
  • Se encarga de documetar cada modelo de cada marca y sus diferentes varintes
  • Cada mercado require diferente informacion acerca de los vehiculos

Our Microservice Application

Meta Data System

Manages all meta data for the whole system.

  • Springboot application
  • Aurora DB
  • SQS Asynchronous Communication
  • Scheduled Jobs

Ext System Mgr

Management of data coming from external systems

  • Springboot application
  • Aurora DB
  • SQS Asynchronous Communication
  • Scheduled Jobs

Auth System

Authorizes and authenticates all login user requests.

  • Springboot application
  • Aurora DB
  • Communication with External Security System

Files System

Manages all kind of self generated or user-uploaded files (pdfs, tiffs, imgs, etc.)

  • Springboot application
  • AWS S3 Filesystem

Additional Systems

  • Bamboo Deployment System
  • Jira Ticket System
  • Confluence Documentation System
  • Bitbucket Repository
  • Pivotal Cloud Foundry Application Platform
  • Amazon Cloud

Summary
